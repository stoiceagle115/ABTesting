A/B Testing and Eyetracking

Evaluating different UI designs and user responses

About

This goal of this project was to compare different UI designs for the same website. These designs were tested on different users using an eye tracking software. The data collected from those tests could be used to qualitatively compare our UI designs.


Layouts

We created two different UI designs with completely opposite design philosophies.
The first soley focused on long vertical and horizontal shapes, while the second focused on more uniform and box-like shapes.

Observations

We tested two users on our UI designs.
An eyetracking software was used to collect data from these test. Some of the results can be seen below.

Key Observations

- Two out of the three users were able to use the interface without the need of outside assistance
- Most of the users took a similar amount of time (around 2 min 35 sec) to finish using the interface
- The help button on the interface seemed to do nothing when one of the users pressed it, and the user had to physically go and ask a worker for help
- All users were immediately able to understand how to use the interfaces' scanner.

Questions

  • 1. What was the first thing that your eyes were drawn to when you saw the interface?
  • 2. Are there any features of a checkout with a store clerk that you would want the self-service kiosk to include?
  • 3. Did it take longer, shorter or the the expected amount of time to use?
  • 4. Was there anything about the interface which caused you great frustration?
  • 5. Were the phsyical and digital aspects of the interface well-integrated?

Results

Our results did not quite match our expectations for the horizontal layout because we were thinking that if everything is displayed on one screen that the user would have looked at several items. Instead, it seems like they really only looked at one or two items. For our vertical nav bar design, the results were also a little concerning because it seemed that the viewer did not pay much attention to the “cakes” and “hot drinks” collapsible drop downs. However, this may have been because they knew these components weren’t actually interactive, and viewers clearly looked at all of the food items on the screen, so our hypothesis that a collapsible drop downs would decrease clutter and focus the user was supported.
